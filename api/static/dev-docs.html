StellarX-402

Developer documentation

Run, integrate, and extend the OpenClaw-compatible executor — x402, Soroban, A2A, and signed Docker jobs.

Getting Started

Prerequisites

  • Python 3.10+
  • Docker installed and running
  • A Stellar testnet account — generate one at lab.stellar.org

Installation

git clone https://github.com/nirmalplays/Stellar-x402.git
cd Stellar-x402
pip install -r requirements.txt

Configure environment

cp .env.example .env

Then open .env and fill in:

Variable Required Description
DEPLOYER_PUBLIC_KEY✅Stellar account that owns the registry contract
DEPLOYER_SECRET✅Secret key for the deployer account
EXECUTOR_PUBLIC_KEY✅Account that receives payments
EXECUTOR_SECRET✅Secret key for the executor account
REGISTRY_CONTRACT_ID✅Deployed Soroban registry contract ID
PUBLIC_BASE_URL✅Your public URL — required for discovery
X402_FACILITATOR_URL❌Default: https://x402.org/facilitator
X402_PRICE❌Price per job in USDC. Default: 0.01
REGISTRY_BYPASS_DEV❌Set true for local dev without a contract

Run the server

# Direct
python -m uvicorn api.main:app --host 0.0.0.0 --port 8000

# Docker Compose
docker compose up --build

Dashboard: http://localhost:8000/ · Marketing: /home

Authentication & Payment

Payment is authorization. No API key needed.

Every request to /execute or /execute/stream must include a payment header. The server verifies payment before running any job.

Option A — x402 v2 (USDC, recommended)

POST /execute/stream
Header: X-Payment: <PaymentPayload JSON>

Facilitator options

Facilitator Network API key needed
https://x402.org/facilitatorTestnetNo
https://channels.openzeppelin.com/x402/testnetTestnet + MainnetYes

Compatible wallets: Freighter, Albedo, Hana, HOT, Klever, OneKey

Option B — Legacy XLM

Send 0.05 XLM directly to EXECUTOR_PUBLIC_KEY, then pass the transaction hash:

POST /execute/stream
Header: X-Stellar-Payment-Tx: <transaction_hash>

Prepare an unsigned transaction first:

POST /api/x402/prepare-payment
Content-Type: application/json

{ "source_public_key": "G..." }

Execution

Basic request body

{
  "agent_id": "agent_402",
  "task": "Run the container and print output",
  "input": {},
  "image": "python:3.11-slim",
  "cmd": "python -c \"print(2+2)\""
}

The API model requires input (use {} if empty). Optional validation flags go under input (e.g. expected_substring).

Full request (common fields)

{
  "agent_id": "agent_402",
  "task": "Describe what this run must accomplish",
  "input": {
    "expected_substring": "done"
  },
  "image": "python:3.11-slim",
  "cmd": "python -c \"print('done')\"",
  "network_enabled": false,
  "secrets": {
    "MY_API_KEY": "sk-123..."
  }
}
Field Type Req Description
agent_idstring✅Registered Soroban agent ID
taskstring✅Task description (AI validation + signed)
inputobject✅May include expected_substring and other validators
imagestring?❌*Docker image; omit / auto → server infers allowlisted image from task + cmd
cmdstring✅Command inside the container
network_enabledboolean❌Default false. true for browser / outbound HTTP
secretsobject❌Env vars for the container; stripped from signed result

* Image is optional when auto-selection is enabled (dashboard default).

Success response

{
  "job_id": "abc-123",
  "status": "completed",
  "output": "4\n",
  "verified": true,
  "signature": "ed25519...",
  "pubkey": "GXXX...",
  "validation_strategy": "deterministic",
  "validation_reason": "Output contains expected substring",
  "signed_payload": { },
  "timestamp": "2026-04-12T10:00:00Z",
  "executor_agent": "openclaw"
}

Status values

Status Meaning
completedJob finished, output validated
failedJob ran but validation did not pass
timeoutContainer exceeded time limit
canceledCanceled by client when supported

Streaming (SSE)

Use /execute/stream. Each data: line is a JSON object.

  • Log lines: {"line": "..."} — stdout and status messages.
  • Mid-run: optional objects such as {"job_id": "...", "phase": "job_accepted"}.
  • Terminal result: full JobResult JSON with job_id, status, signature, etc.
data: {"line": "Task started..."}

data: {"job_id":"…","status":"completed","verified":true,...}

Browser automation (Playwright)

{
  "image": "mcr.microsoft.com/playwright/python:v1.45.0-jammy",
  "cmd": "python3 script.py",
  "network_enabled": true
}

Memory limit is raised to 512MB when network_enabled: true or when the image name contains playwright.

Agent Identity (EIP-8004)

Discovery endpoints

GET /.well-known/agent-card.json         → A2A Agent Card
GET /.well-known/agent-registration.json → EIP-8004 registration

Both return 503 if required env vars are not set (PUBLIC_BASE_URL, REGISTRY_CONTRACT_ID for registration). Nothing is fabricated.

Soroban identity format

agentRegistry: stellar:{NETWORK}:{REGISTRY_CONTRACT_ID}
agentId: agent_402   ← Soroban string ID, not an ERC-721 tokenId

A2A Protocol

Send a task (HTTP+JSON)

POST /message:send
Header: X-Payment: <payment>
Content-Type: application/json

{
  "agent_id": "agent_402",
  "task": "docker",
  "input": {
    "image": "python:3.11",
    "cmd": "print(2+2)"
  }
}

JSON-RPC

POST /a2a/jsonrpc

Methods: SendMessage | GetTask | ListTasks | CancelTask

Limitations

  • Tasks are stored in-memory only — lost on restart.
  • SendStreamingMessage and push notifications are not implemented — use /execute/stream for native SSE.
  • In-flight cancel is not implemented.

Validation

Strategies are chosen from your request and output:

Strategy Trigger How it works
deterministicexpected_substring in inputSubstring must appear in stdout
rule_basedNo substring; rules applyChecks patterns such as [ERROR] / [TIMEOUT]
ai_basedWhen AI validation is enabledLLM evaluates stdout vs task description

On-Chain Registry

Deploy the contract

cd contracts/registry
stellar contract deploy \
  --wasm target/wasm32-unknown-unknown/release/registry.wasm \
  --network testnet

Register your agent

python scripts/registry_client.py

Publish metadata to IPFS (optional)

# Set PINATA_JWT in .env first
python scripts/publish_agent_metadata_ipfs.py

Secrets handling

Secrets are injected as environment variables into the container but are stripped from the signed result before returning to the caller.
